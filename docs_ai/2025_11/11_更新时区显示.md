目的

修复 SystemLocalTimePage 所选时区显示不正确的问题，并新增明确的 UTC 时间显示。

修改点

- 在 `lib/page/system/settings/system_local_time_page.dart` 中：
    - 确保 `timezone` 数据库（tzdata）初始化（`tzdata.initializeTimeZones()`）。
    - 使用 `timezone` 库的 `tz.getLocation()` 与 `tz.TZDateTime.now(location)` 返回指定 IANA 时区的墙钟时间，而不是对本地时间进行错误转换。
    - 添加对显式 `UTC` 关键字的处理，直接显示 `DateTime.now().toUtc()` 的格式化字符串。
    - 保持 LocalTimeStore 单例逻辑不变（存储时区字符串）。

验证步骤

1. 运行应用并打开 「本地时区设置」 页面（`SystemLocalTimePage`）。
2. 在输入框输入 `Asia/Shanghai` 并保存，页面右侧应显示当前 `Asia/Shanghai` 的当地时间（例如：如果 UTC 为 08:00，北京时间应为
   16:00）。
3. 清除输入后，页面应回退到设备本地时间显示。
4. 输入 `UTC` 并保存后，页面应在 “当前 UTC 时间” 与 “使用的时区及显示时间” 中都显示相同的 UTC 时间。

注意

- tz 数据库初始化在 `_ensureInit` 中做一次；若初始化失败会回退至设备本地/UTC 时间显示。
- API 时间仍应按项目约定：与后端交换使用 UTC（`toUtc()`/解析为 UTC），UI 显示使用 `.toLocal()` 或指定时区的 `TZDateTime`。
